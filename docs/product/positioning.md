# Positioning

**Status:** Decided

## The line

> The non-linear editor for thought. A studio for your ideas.

## Against the category

Most note-taking apps are organised around **storage** — where you put things. Fosta is organised around **flow** — what you do with things.

| Category thinking | Fosta thinking |
|---|---|
| Where does this note go? | What stage is this idea at? |
| Which folder? | Which lens am I using right now? |
| Organise first, work later | Capture now, organise when you're ready |

## The DaVinci Resolve analogy

DaVinci Resolve separates editing stages into pages — Cut, Edit, Fusion, Color, Fairlight, Deliver. Each page is a different mode of working on the same project. Switching pages doesn't change the project — it changes your relationship to it.

Fosta applies this to thought. Each view is a different mode of working with your ideas. The notes don't change — your relationship to them does.

## Target user

Creative professionals who:
- Work with ideas across multiple contexts (home, work, on the go)
- Think non-linearly but need to produce linear outputs
- Have tried Notion, Obsidian, Bear, and found them either too structured or too unstructured
- Care about data ownership and don't want their thinking trapped in a proprietary format

## Competitive positioning

See `docs/engineering/` for the full competitive landscape. High-level:

- **vs Obsidian:** More opinionated workflow, better publishing, no plugin complexity
- **vs Notion:** Local-first, offline, plain text — no lock-in
- **vs Bear:** Richer structure (canvas, timeline), pipeline workflow
- **vs Capacities:** Flat tags not type schemas — organise by what something is, not where it lives
