# Fosta — Product Overview

**Status:** Decided — Final  
**Version:** v0.2.0

## What it is

Fosta is a macOS-native, local-first, offline-first note-taking and idea-development app built around a pipeline metaphor. It is inspired by DaVinci Resolve's page-based layout.

## The thesis

> Notes flow through a pipeline — you capture fast, sort later, develop ideas in layers, sequence them, then publish.

## Positioning

> The non-linear editor for thought. A studio for your ideas.

## Target audience

Creative professionals: filmmakers, designers, animators, multi-disciplinary thinkers. People who work with ideas across multiple contexts and need a tool that matches how creative thinking actually works — non-linear, iterative, context-switching.

The creator (Ben) is the primary target user.

## Domain

**fosta.studio** — final, do not suggest alternatives.

## The pipeline structure

| Stage | View | What happens |
|---|---|---|
| Capture | Capture (global) | Fast intake — notes always land in Inbox/ |
| Triage | Sort | Process inbox, apply tags, create structure |
| Develop | Layer | Deep work — block editing, backlinks, comments |
| Sequence | Track | Arrange notes into timelines or spatial maps |
| Publish | Share | Compose and publish from note references |

## The core insight

All views show the same data through different lenses. The user's mode of working determines which view they're in — not which data they can access. There is one vault, one set of notes, one data model. Views are different ways of working with it.

## What makes it different

- **Local-first, files on disk** — notes are plain Markdown files. No lock-in, always accessible.
- **Pipeline metaphor** — not a folder hierarchy, not a graph. A flow.
- **Creative professional focus** — designed for how filmmakers, designers, and animators think, not for GTD productivity workflows.
- **macOS native** — built with Tauri 2, feels like a real Mac app.
